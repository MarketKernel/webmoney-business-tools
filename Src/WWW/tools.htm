Главная::Инструменты

На данной странице приведен список наиболее распространенных задач, для решения которых, достаточно  базовых инструментов. Для решения более сложных задач применяются бизнес-автоматы.

Новые инструменты WebMoney Business Tools версии 2.0

Работа с доверенными кошельками

  • Добавление кошелька вручную (когда нет прав на его просмотр).
  • Запрос доверия на совершение операций с кошелька клиента через SMS/USSD.
  • Установка ключа мерчанта (для создания платежных ссылок и получения данных об оплате и пр. когда нет доверия на кошелек).

Работа с операциями по доверию

  • Платеж от клиента по SMS/USSD.
  • Ссылка для оплаты через WM Merchant.
  • Просмотр деталей операции, принятой через WM Merchant.
  • Массовые платежи (см. ниже описание).
  • Возможность оффлайн-поиска по базе данных.
  • Добавлен график для визуализации данных. По клику на графике в форме, появится альтернативный увеличенный вид.
  • Экспорт в открытый формат xlsx (поддержка Libre Office и пр.).

Массовые платежи

  • Импорт/экспорт в формат XML https://masspayment.webmoney.ru
  • Живое отображение прогресса проведения.
  • Экспорт результата в Excel (xlsx).
  • Просмотр результата проведения.

Просмотр аттестационной информации

  • Возможность оффлайн-поиска.

Работа с контрактами

  • Возможность создать контракт.
  • Возможность просмотреть контракты (список хранится оффлайн).
  • Возможность обновить количество подписантов.

Верификация клиентов через X19

  • 8 способов платежа, для каждого своя форма с предварительной валидацией данных.

Настройки

  • Возможность задать формат номера запроса (UnixTimeStamp или формат yyMMddHHmmssfff (2 цифры года, месяц, день, часы, минуты, секунды и миллисекунды, склеенные в одну строку)). Возможность задать смещение.
  • Добавились цвета для операций, счетов, контрактов и пр.
  • Изменение ключа.
  • Изменение пароля.
  • Изменение подключения к базе данных.
  • Язык: поддерживается английский и русский.

Отчеты об ошибках

  • Можно отправить отчет разработчику, прямо из окошка ошибки. Есть возможность дописать свой комментарий. Отчеты доходят и читаются (если не удалось отправить – вы получите сообщение).
  • Добавлены логи в формате log4net.

Прочее

  • При выборе кошелька, теперь указывается не просто его номер, но и название и сумма.
  • Возможность написать разработчику из программы (не используется WM-почта).
  • Возможность входить без пароля.
  • Добавлена возможность сохранять данные в базу данных (операции, контракты, данные сертификата, доверенности и пр.).
  • Выбор базы данных: сейчас MS SQL или SQL Compact Edition (по умолчанию).
  • Работа в Ubuntu в тестовом режиме (в данной версии не поддерживается подключение к базе данных).
  • Возможность подключения плагинов без переустановки программы.
  • Возможность заменять стандартные окна на альтернативную реализацию с помощью плагинов.
  • Программа переведена на русский и английский: при входе учитывается язык системы.

Набор инструментов WebMoney Business Tools версии 1.0

1. Работа с проблемными операциями: (1) бескомиссионный возврат средств, (2) отправка сообщения на WM Keeper без авторизации.

Пример: клиент потребовал возврат средств, в соответствии с правилами сервиса, написав сообщение на Keeper или e-mail. Программа поможет выяснить проблему (написать сообщение без авторизации), при необходимости, вернуть средства без уплаты 0.8% комиссии системы.

2. Совместное управление кошельками по доверию.

Практические примеры:

  • Совместный интернет-магазин (распределенный контроль поступлений, учет прибыли и пр.). Применимо в простейших случаях, когда не нужны все возможности сервиса Capitaller.
  • Оказание технической поддержки сторонним сервисам. Сервис, который оказывает тех. поддержку, должен получить разрешение на просмотр операций клиентов.
  • Контроль инвестиционных средств или контроль продаж через посредника (в тех случаях, когда посредник получает процент от прибыли).

3. Просмотр операций в системе за произвольный период (с указанием общей суммы дохода и расхода). Возможность экспорта в Excel.

Практические примеры:

  • Анализ продаж и вычисление необходимого количества закупки/заказа товара (в т.ч. цифрового товара или финансовых инструментов).
  • Налоговая отчетность (после экспорта операций в Excel, необходимо получить операции вывода средств из системы за определенный период и отобразить их в отчете).

4. Просмотр аттестационной информации клиента. Просмотр бизнес-уровня (BL), уровня доверия (TL), наличия отзывов в арбитраже.

Практические примеры:

  • Принятие решения о выдаче кредита. Более простой и удобный доступ ко всем скрытым данным клиента, возможность отправки сообщения на Keeper, телефон и счета на оплату.
  • Поиск бизнес-партнеров.

5. Ручная обработка операций: перевод средств (возможность протекции), ввод кода протекции.

Практические примеры:

  • Дилеры системы WebMoney, использующие такую схему работы: передача наличных средств клиенту при личной встрече, после получения от клиента кода протекции платежа.
  • Простые интернет-магазины, не владеющие средствами автоматизации для приема платежей.

6. Ручная обработка счетов: выписка счета, оплата счета (возможность протекции), проверка оплаты счетов.

Пример: простая схема продажи товаров (возможно в кредит).

7. Работа со своим кошельком по доверию (доверие на второй WMID), с целью обеспечения безопасности.

Практические примеры:

  • Доверить второму WMID только просмотр операций рабочего WMID и вывод прибыли по строго установленным лимитам (к примеру, раз в месяц). Ключи рабочего WMID хранить в сейфе.
  • Использование устройства для безопасного хранения ключей, не поддерживающего работу с WM Keeper Classic, но поддерживающего работу со стандартными ключами WM Light (пример такого устройства — uaToken). В таком случае, ключи от рабочего WM Classic можно уничтожить, а ключи от Keeper Light хранить в токене без возможности экспорта. Доступ к средствам, операциям и исходящим счетам WM Classic получать по доверию.

 

Кроме этого, ведется работа над новыми универсальными инструментами, которые позволят производить массовые выплаты, массовые выписки счетов, рассылки новостей клиентам, работать со счетами Telepat и не только. Эти инструменты будут доступны в новой версии программы.
